# .planning/ — 任务规划文件目录

所有 AI Agent（Claude、Cursor、Codeium）在执行重要开发任务前，必须在此目录创建规划文件并等待用户批准。

## 命名规范

```
{YYYYMMDD}_{task_name}.md
```

示例：
- `20260324_auth_module.md`
- `20260401_data_pipeline_v2.md`
- `20260415_refactor_model_backbone.md`

## 文件格式模板

```markdown
## 任务：[具体任务名]

**背景**：[为什么要做这件事，与项目目标的关联]
**影响范围**：[会修改或创建的文件列表]
**前置条件**：[依赖哪些已完成的工作]

### Stage 1: [阶段名]
- **目标**：[具体交付物]
- **成功标准**：[可验证的结果]
- **状态**：Not Started

### Stage 2: [阶段名]
- **目标**：...
- **成功标准**：...
- **状态**：Not Started

**待确认事项**：
- [ ] 问题一
- [ ] 问题二
```

## 生命周期

1. Agent 创建规划文件 → 等待用户批准
2. 用户批准后 → 开始执行，更新各 Stage 状态
3. 完成后 → 追加 `## 完成记录`
4. 历史文件永久保留，不删除

## 何时需要规划文件

- 创建新的源代码文件
- 修改核心架构组件
- 新增重要功能模块
- 多步骤任务（≥ 3 步）
- 架构层面的设计决策

单行 bug fix、注释修改、配置调整**不需要**规划文件。
